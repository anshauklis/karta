# Configuration

All configuration is done via the `.env` file in the project root.

## Required Variables

Auto-generated by `install.sh`. Only set manually if not using the script.

| Variable | Description | Generation |
|----------|-------------|------------|
| `JWT_SECRET` | Signing key for JWT tokens | `openssl rand -base64 32` |
| `CONNECTION_SECRET` | AES-256-GCM key for encrypting DB passwords | `openssl rand -base64 32` |
| `POSTGRES_PASSWORD` | Internal PostgreSQL password | Random 24-char alphanumeric |

:::{warning}
Changing `CONNECTION_SECRET` after initial setup will make existing encrypted database passwords unreadable. You will need to re-enter passwords for all connections.
:::

## Resource Limits

Default memory limits in `docker-compose.yml`:

| Service | Memory Limit | Adjust If... |
|---------|-------------|--------------|
| **postgres** | 1 GB | Large number of concurrent queries |
| **api** | 4 GB | Complex Python code charts or large datasets |
| **frontend** | 768 MB | Usually sufficient |
| **nginx** | 256 MB | Usually sufficient |
| **redis** | 256 MB | Large query cache needs |

To adjust, edit `mem_limit` in `docker-compose.yml`:

```yaml
api:
  mem_limit: 1g
```

## Redis Cache

Redis is optional but recommended for production:

- Query results are cached for 5 minutes (SHA-256 key based on query + connection + filters)
- Repeated dashboard views are significantly faster
- If Redis is unavailable, the application falls back gracefully to no caching

To disable Redis, remove the redis service from `docker-compose.yml` and unset `REDIS_URL`.
